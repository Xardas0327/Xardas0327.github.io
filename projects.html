Published

This is a cooperative game, in which you have to get out of a labyrinth filled with enemy robots. It was created with Unity and it uses Daydream (Google vr) and ARCore. It was included in the Daydream store, which means it meets all of the Daydream App Quality criteria. You can watch the trailer here.

The game only works through WiFi, and both players have to be on the same WiFi network (or direct routing has to be set up between the networks). One of the players is a prisoner, who plays in virtual reality (VR) mode, and the other one is a hacker, who plays in normal mobile mode or augmented reality (AR) mode.

The prisoner has to escape the labyrinth within the time limit, while avoiding enemy robots and following the instructions of their partner. The game requires Cardboard or Daydream and their associated services to play in VR mode. The game can be played in a standing position, but you can also play with Daydream while sitting.

The hacker sees the timer, the health of the prisoner, the map of the labyrinth and whether the selected robot is active or not. The hacker has the ability to shut down a robot for a short time, but this ability has a 30 second cooldown. The software needs access to the camera of the phone in order to function properly in AR mode. Or you can play without AR.

The enemy robots will attempt to attack the prisoner, and if they notice them, they will recharge significantly faster. However, if the hacker disables the robot, it will forget that it ever saw the prisoner.

The Meeting Room is more an interesting feature than fun. There is an empty room, where the two players can meet with each other. It requires all of the VR and AR prerequisites. This feature is currently in beta.

Unfortunatelly, I had to stop this project, because of Google's shutdown of the Daydream platform. Moreover, Google doesn't support Daydream on Android 11, so it is not 100% that the game will work. (It works on Pixel 2.) It was available in Google Play Store until 2023-08-24.

Learning Projects

This is my first game, which I created with Unity. It is a "ball in a maze" game, which was implemented for mobile phones.

Game description: The point of the game is that you have to move the ball to a point (yellow light) by map rotate (phone rotate). The game is complicated by the fact that the mazes have holes and/or some walls are missing. There is a check point in the center of every maze, but you have only 3-5 "lives". So if you lose them you have to start the current map at the beginning.

If you complete a maze, the system unlocks the next one. But you can play with them several times, because you can collect stars from all of the 8 maps. You get maximum 3 starts on a map:

  • when you completed the map
  • if you didn't lose any tries
  • if you do it in time

I don't feel it is a good enough game, that's why I didn't put it into Google Play Store. But I am sharing the android apk file here.
Requirements: Android 5.0 with accelerometer
Recommended: Android 5.0 with gyroscope (You can change it in the options.)

That was one of the reasons that the mod creators started to rewrite it from the begining.

I participated in a Game Jam. There were 9 people and we had 2 days to create a 2D tower defense game in Unity. This game is playable on Windows PC and Android. I wrote the castle, which you have to defend, and the enemy spawn system. In addition I designed the last level.

Game description: You can create one tower at the beginning on every level. When you kill the enemy, you will get gold to buy other towers or gems (they are the boosts).

Type of gems:

  • Damage (red)
  • Fire rate (green)
  • Slow-down enemy (blue)
  • Bullets shoot range (purple)
  • Bullets explode range (yellow)

We didn't upload it to any store, but I am sharing the android apk file here.
