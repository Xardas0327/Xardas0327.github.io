This project is a user interface for Unity ML-Agents. It helps the users, that they can create/modify a yaml file inside Unity Editor. Moreover, the users can run the mlagents-learn command without command line.

It is available on Github and Unity Asset Store

Requirements:

  • Unity ML-Agents Release 20, 21 and 22.
  • OS:
    • Windows (Tested: Windows 11)
    • Ubuntu (Tested: Ubuntu 20.04)
    • MacOS (Tested: MacOS 13, 14, 15)
  • Unity 2021.3.30f1 or later (It is depend on ML-Agents Release)

Features:

  1. Creation
    A user can create parts of the ML-Agents config yaml file as ScriptableObject. They contains tooltips, validation. Futhermore, the users can see only the actual fields. For example: If PPO Trainer Type is selected, the specific variables of SAC type is not visible.
  2. Import
    A user can import any ML-Agents config yaml file from samples or their own custom project. The files will be created in Assets/Xardas/ML-Agents Configuration/Files folder. (If the folder doesn't exist, it will be created.)
    Limit: If the yaml file has default_settings, it will be part of the Behaviors. After that, if the users want to modify the same variable in the Behaviors, they have to update them one by one.
  3. Export
    A user can export ML-Agents Config ScriptableObjects into a yaml file.
  4. Run Command Line
    Firstly, it has to be configureated. It is in Project Settings/ML-Agents Configuration. The users can use Python Virtual Environment, if they want.
    There are 3 options:
    • None: The user don't use Virtual Environment.
    • Basic Python: The Virtual Environment, which the Python can create by default. The users have to add the activate file.
    • Anaconda: If the users use the Anaconda package management, they have to add the activate file and the name of Virtual Environment.

I created this 2D side scroller platformer game with a small independent team by Unity for a Game Jam, where I was a developer.

The player is a thief, which is not so good, but the gods give him a chance. If the player bring back (steal) artifacts from a museum, Ammut won't eat their heart (Egypt Mythology).
There are 16 artifacts on the map, but the player has to collect only 12. (There are 4 different artifacts.)
It is not too easy, because the player has to jump on platforms. If the guards catch them, they will lose 5 years in their life. The age start from 20 and the game end, when the age is 70. Moreover, the artifacts can spawn 27 places, so they are not always on the same place. Futhermore, there are 3 doors (blue, yellow, red), which can be opened by their cards only and the time is ticking. Last, when the player is 40, they will become bigger and a bit slower and when they are 60, they will become a bit more slower.

This is a cooperative game, in which you have to get out of a labyrinth filled with enemy robots. It was created with Unity and it uses Daydream (Google vr) and ARCore. It was included in the Daydream store, which means it meets all of the Daydream App Quality criteria. You can watch the trailer here.

The game only works through WiFi, and both players have to be on the same WiFi network (or direct routing has to be set up between the networks). One of the players is a prisoner, who plays in virtual reality (VR) mode, and the other one is a hacker, who plays in normal mobile mode or augmented reality (AR) mode.

The prisoner has to escape the labyrinth within the time limit, while avoiding enemy robots and following the instructions of their partner. The game requires Cardboard or Daydream and their associated services to play in VR mode. The game can be played in a standing position, but you can also play with Daydream while sitting.

The hacker sees the timer, the health of the prisoner, the map of the labyrinth and whether the selected robot is active or not. The hacker has the ability to shut down a robot for a short time, but this ability has a 30 second cooldown. The software needs access to the camera of the phone in order to function properly in AR mode. Or you can play without AR.

The enemy robots will attempt to attack the prisoner, and if they notice them, they will recharge significantly faster. However, if the hacker disables the robot, it will forget that it ever saw the prisoner.

The Meeting Room is more an interesting feature than fun. There is an empty room, where the two players can meet with each other. It requires all of the VR and AR prerequisites. This feature is currently in beta.

Unfortunatelly, I had to stop this project, because of Google's shutdown of the Daydream platform. Moreover, Google doesn't support Daydream on Android 11, so it is not 100% that the game will work. (It works on Pixel 2.) It was available in Google Play Store until 2023-08-24.

If you complete a maze, the system unlocks the next one. But you can play with them several times, because you can collect stars from all of the 8 maps. You get maximum 3 starts on a map:

  • when you completed the map
  • if you didn't lose any tries
  • if you do it in time
